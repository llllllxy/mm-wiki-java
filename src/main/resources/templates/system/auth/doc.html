登录认证方式配置文档

  • LDAP 登录
  • HTTP 登录

认证 URL 格式如 ldap://ldap.server:389 或 ldaps://ldap.server:636
附加参数 
{
  "basedn": "dc=example,dc=com",
  "bind_username": "readonly",
  "bind_password": "password",
  "account_pattern": "(&(objectClass=User)(userPrincipalName=%s))",
  "given_name_key": "displayName",
  "email_key": "mail",
  "mobile_key": "mobile",
  "phone_key": "telephoneNumber",
  "department_key": "department",
  "position_key": "Position",
  "location_key": "physicalDeliveryOfficeName",
  "im_key": "im"
}
注意事项 附加参数必须是合法 JSON，并且字段值与实际 LDAP 环境匹配。
认证 URL 使用支持 POST 的 HTTP/HTTPS 接口，例如 https://example.com/login
请求参数 
username=xxx
password=xxx
ext_data=xxx
返回格式 
{
  "message": "",
  "data": {
    "given_name": "张三",
    "mobile": "13800000000",
    "phone": "010-12345678",
    "email": "user@example.com",
    "department": "研发部",
    "position": "工程师",
    "location": "B座 3 层",
    "im": "wechat"
  }
}
失败时 `message` 返回错误原因，`data` 可为空。
